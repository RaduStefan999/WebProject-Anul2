1. Frontend Architecture

1.1 Pages

Routes :
/main -- Main Page on which the user first arives
/courses -- Courses page where a logged in user can see his progress or choose a new course
/course/{course_name} -- Course page where user can study course
/mygarden -- My garden page where user can manage his own garden
/leaderboard -- Leaderboard page where users can see the best performing users of the application
/managecourses -- Manage courses page accessable only by admins where admin can add, edit or remove course

The frontend application uses fontawesome in order to take advantege of the icons it provides.
The frontend application uses only vanilla css and javascript.

2. Backend Architecture

2.1 Frontend-Backend Communication

The backend is implmeneted in nodeJS and it will provide a REST API for the front-end to communicate with.
The frontend pages will be generated in a hibrid fashion :
For static pages like the front page where the only information required from the backend is the number of users and the number of courses, the page will be generated by the backend enierly.
For more dynamic pages like My Garden page, the frontend will use AJAX calls to the backend REST API in order to obtain data.
For authentication, the backend will generate a token for each logged in user that will be received by the frontend and stored in local storage. The token will be provided for each API call that requires authentication, for example, /get_my_garden_info.

2.2 Data Flow

The data will be sent and received in JSON format. The backend will provide data for MyGarden, All Courses, a Specific Corse and The Leaderboard.
The frontend will do POST request with data such as User Registration Info, User Login Info, Feedback, Add Course Info, Edit Course Info, Add Plant To Garden Info.

2.3 Code structure

The backend follows the MVC pattern, the authorisation will be structured in middlewares and each route will be managed by a function in a controller.

2.4 Data storage

The data will be stored either in a MySQL database or and Oracle database with a layer of PL/SQL depending on how much time we have.
